탐색적 데이터 분석

Exploratory Data Analysis

  • 탐색적 데이터 분석(EDA)는 존 튜키라는 미국의 저명한 통계학자가 창안한 데이터 분석 방법론
  • 기존의 통계학이 정보의 추출에서 가설 검정등에 치우펴 자료가 가지고 있는 본연의 의미를 찾는데 어려움이 있어 이를 보완하고자 주어진 자료만 가지고도 충분한 정보를 찾을 수 있도록 여러가지 탐색적 자료 분석 방법을 개발

탐색적 데이터 분석의 필요성

  • 데이터의 분포 및 값을 검토함으로써 데이터가 표현하는 현상을 더 잘 이해하고, 데이터에 대한 잠재적인 문제를 발견
  • 본격적인 분석에 들어가기에 앞서 데이터를 다시 수집하거나 추가로 수집하는 등의 결정을 내릴 수 있음
  • 데이터를 다양한 각도에서 살펴보는 과정을 통해 문제 정의 단계에서 미처 발생하지 못했을 다양한 패턴을 반결하고, 이를 바탕으로 기존의 가설을 수정하거나 새로운 가설을 세울 수 있음
  • 데이터에 대한 이런 시직은 이후에 통계적 추론을 시도하거나 예측 모델을 만들 때 유용하게 사용

탐색적 데이터 분석 과정

  • 문제 정의 단계에서 세웠던 연구 질문과 가설을 바탕으로 분석 계획을 세우는 것
  • 분석 계획에는 어떤 속성 및 속성 간의 관계를 집중적으로 관찰해야 할지, 이를 위한 최적의 방법은 무엇인지가 포함되어야함
  • 분석의 목적과 변수가 무엇이 있는지 확인하고, 개별 변수의 이름이나 설명을 가지는지 확인
  • 데이터를 전체적으로 살펴보기
    • 데이터에 문제가 없는지 확인
      • 데이터의 중복(동일 데이터가 여러 번 기록되었는가)
      • 불균형(특정 클래스의 데이터가 현저히 많은가)
      • 타입의 불일치(데이터의 예상 타입과 다른 데이터가 있는가)
      • 스케일 차이(각 변수의 값의 스케일 차이가 심한가)
      • 불일관성(동일한 데이터가 서로 다른 형식으로 기록된 경우 일관되게 조치)
      • 변동성(시간에 따라 급변하는가)
    • head나 tail 부분을 확인
    • 추가적으로 다양한 탐색(이상치, 결측치 등을 확인하는 과정)
  • 데이터의 개별 속성값을 관찰
    • 각 속성 값이 예측한 범위와 분포를 갖는지 확인
      • 예시) 따릉이 사용량 데이터 → 기온이 30도 이상일 때 사용량이 급감할 것이다 예상했으나 실제 데이터에서 기온이 30도가 넘은 날에도 사용량은 지속적으로 증가하였다
    • 만약 그렇지 않다면, 이유가 무엇인지를 확인
      • 예시) 최고 기온 시간대를 제외한 나머지 시간대에 사용량이 줄어들지 않았고 또한 단거리 사용량이 많다보니 기온이 사용량 변화에 큰 영향을 주지 않는 것으로 확인하였다.
  • 속성 간의 관계에 초점을 맞추어, 개별 속성 관찰에서 찾아내지 못했던 패턴은 발견(상관관계, 시각화 등)

탐색적 데이터 분석 단계

  • 문제 정의(질문, 가설)
    • 현재 우리가 가진 문제 상황이 무엇인지 정의하고 왜 이런 문제가 발생했는지 질문하고 문제의 원인에 대한 가설을 세운다
  • 분석 계획
    • 문제 해결을 위해서 어떤 데이터를 어떻게 분석할 것인지에 대한 계획 수립
  • 데이터 관찰
    • 정의된 데이터에 대해 다각도에서 관찰
  • 재현
    • 관찰한 데이터를 기반으로 문제 상황 재현
  • 요인 추가 / 가설 변경
    • 재현된 상황에서 도출된 정보를 바탕으로 새로운 요인을 추가하고 가설을 상황에 맞게 변경
  • 실험

연구 질문

  • 연구 과정은 방법론을 선택하는 것에서부터 보고에 이르는 많은 다른 단계를 수반
  • 종종 간과되는 연구 과정의 영역은 연구 질문의 개발이며, 이과정은 매우 중요한 과정
  • 왜냐하면 올바른 질문을 하지 않는다면, 정답을 얻을 수 없을 것이기 때문
  • 목적이 본질적으로 질적인지 양적인지에 따라 어떤 유형의 연구 문제를 활용해야하는지가 결정
  • 좋은 연구 질문의 요소
    • non-biased : 편향되지 않는 질물
    • focused : 초점이 잡힌 질문
    • pertinent : 적절한 질문
    • intriguing : 흥미로운 질문
    • relevant : 관련이 있는 질문
    • answerable : 답변이 가능한 질문

정량적 질문 유형(Quantitative Question Type)

  • 정량적 연구는 일반적으로 어떻게, 무엇을, 언제, 어디서를 결정하는데 사용
    • 정량적 연구란 무엇인가?
      • 측정 가능한 데이터를 체계적으로 수집하고 분석하는 강력한 연구 방법론
      • 엄격한 통계 및 수학적 기법을 통해 구조화된 설문조사, 통제된 실험 또는 기타 정의된 데이터 수집 방법에서 인사이트를 추출
  • 정량적 연구를 위한 설문지는 질적 질문보다 많은 면에서 개발하기 쉬움
  • 한 번 배워야 할 것을 결정하면 이러한 질문을 객관식, 선택형, 격자형 문제 등에 통합하는 것이 더 직관적이기 때문
  • 더 큰 데이터 집합을 수집하기 때문에 정량적 방법론에는 일반적으로 보다 구체적인 연구 질문이 포함됨
  • 서술형 질문(Descriptive Questions) : 문제의 개념 또는 주제를 설명
    • 예시) 빈도, 하루 중 시간, 사용 목적 등과 같은 제품의 사용을 이해하는 것
  • 비교 질문(Comparative Questions) : 두 그룹, 개념 또는 기타 변수 간의 차이를 분석하는 데 사용
    • 예시) 두 제품 간의 사용 빈도 비교, 남성 대 여성의 브랜드 선호도 등
  • 관계 기반 질문(Relationship-Based Questions) : 인과관계에 기반한 질문들은 한 변수가 다른 변수에 어떻게 영향을 미치는지 이해하는 데 효과가 있음
    • 예시) 색상이 특정 제품을 구매하려는 욕구에 어떻게 영향을 미치는 지

질적 질문 유형(Qualitative Question Types)

  • 질적 연구는 본질적으로 정량화가 덜되고, 목표에 관한 의미를 발견하고, 이해하고, 탐구하는 데 더 초점을 맞춤
  • 질적 연구 질문은 직접적이지 않고 본질적으로 훨씬 모호함
  • 탐색적 질문(Exploratory Questions) : 정량적 연구의 서술형 문제와 유사하게, 선입견으로 결과에 영향을 주지 않고, 무언가를 이해하는 것으로 보림
    • 예시) 제품이 어떻게 사용되는지 또는 특정 주제에 대한 인식을 묻는 것은 두가지 유형의 탐색적 질문
  • 예측 질문(Predictive Questions) : 이러한 질문들은 주제나 행동을 둘러싼 의도나 미래의 결과를 이해하려고 함
    • 예시) 소비자가 왜 특정한 상황에서 행동하는지 또는 어떤 상황이 발생한다면 어떻게 느끼는지 묻는 것이 포함
  • 해석적 질문(Interpretive Questions) : 결과에 영향을 주지 않고 특정 주제 또는 개념에 대한 피드백 수집
    • 예시) 새로운 제품 개념을 테스트하고 전달 요청이 어떻게 해석되는지 이해하는 것은 이 유형에 해당

이상값 발견 기법

  • 개별 데이터 관찰 : 데이터 값을 눈으로 보며 전체적인 추세와 특이사항 관찰
  • 통계값 활용 : 요약 통계 지표(summary statistics)
  • 시각화 활용 : 확률 밀도 함수, 히스토그램, 점 플롯(dot plot), 워드 클라우드, 시계열 차트, 지도 등
  • 머신러닝 기법 활용 : 클러스터링(Clustering) 등을 통해서 이상치 확인
  • 통계 기반 탐지(Statistical-based Detection) : Distribution-based, Depth-based
  • 편차 기반 방법(Deviation-based Method) : Sequential exception, OLAP data cube
  • 거리 기반 탐지(Distance-based Detection) : Index-based, Nested-based, Cell-based, Local-outliers, Partition-based

5가지 숫자 요약(Five-number summary)

  • 데이터 집합에 대한 정보를 제공하는 통계량으로 가장 중요한 표본 백분위수 5가지로 구성 (요약 통계 지표 중 하나)
    • 최대값(maximum)
    • 상위 사분위수(upper quantile) or 제 3사분위수(Q3)
    • 중앙값(median)
    • 하위 사분위수(lower quantile) or 제 1사분위수(Q1)
    • 최소값(minimum)

Descriptive Statistics

  • 데이터 시각화를 통해 더욱 효과적으로 한 눈에 데이터 정보를 탐색할 수 있음

속성 간의 관계 분석

  • Nominal Data(명목) : 선수의 등번호, 출신 지역, 성별
  • Ordinal Data(순서, 서열) : 등수, 소득분위
  • Interval Data(등간) : 온도, 설문조사 결과
  • Ratio Data(비율) : 몸무게, 키, 거리
  • 등간, 비율 데이터는 아래 두가지 속성 중 한가지를 가진다
    • Continuous Data(연속형)
    • Discrete Data(이산형)
  • 어떤 속성의 데이터가 조합되냐에 따라 요약 통계 및 시각화의 방향이 정해지기 때문에 속성 간의 관계 분석이 필요하다
