CNN

Convolutional Neural Network

참고 사이트(https://mijeongban.medium.com/딥러닝-머신러닝-cnn-convolutional-neural-networks-쉽게-이해하기-836869f88375)를 기반으로 학습 내용 정리

CNN이 무엇일까?

  • CNN(Convolutional Neural Network)란 딥러닝에서 주로 이미지나 영상 데이터를 처리할 때 쓰이며 ‘Convolution’이라는 전처리 작업이 선행되는 Neural Network 모델
  • 도입 배경 : DNN(Deep Neural Network)의 문제점에서 출발
    • 일반 DNN은 기본적으로 1차원 형태의 데이터 사용
    • 때문에 이미지가 입력값이 되는 경우 이것을 평탄화(flatten) 시켜서 한 줄 형식의 데이터로 만들어야함
      • 과정 추가 설명
        • 1단계(이미지 준비) : 입력 이미지가 28x28 픽셀 크기의 흑백 이미지일 경우 이미지는 2차원 배열로 표현됨(만약, RGB 컬러 이미지일 경우 높이, 너비, 채널 형식의 3차원 배열이된다)
          image = [[pixel1, pixel2, ..., pixel28],
         [pixel29, pixel30, ..., pixel56],
         ...
         [pixel729, pixel730, ..., pixel784]]
        • 2단계(이미지 정규화 - 선택 사항) : 딥러닝 모델의 학습 효율을 높이기 위해 이미지의 픽셀 값을 0과 1사이로 정규화
          image = image / 255.0
# 각 필셀 값은 0~255 사이의 값을 가지므로 255.0으로 나눠준다
        • 3단계(평탄화 - flatten) : 2차원 배열인 이미지를 1차원 배열로 변환(28x28 크기의 이미지는 784개의 원소를 가지는 1차원 배열로 변환된다)
          flattened_image = [pixel1, pixel2, ..., pixel784]
        • 4단계(모델 입력) : 평탄화된 1차원 배열을 딥러닝 모델의 입력값으로 사용
          model_input = flattened_image

    • 이 과정에서 이미지의 공간적/지역적 정보(spatial/topological information)가 손실되는 문제가 발생
      • 공간적 / 지역적 정보 손실
        • 공간적 정보란 이미지의 각 픽셀 사이의 위치 관계와 패턴을 의미
          • 평탄화가 진행되면 2차원 배열이 1차원으로 바뀌기 때문에 2차원에서의 행과 열 정보가 손실되고 단순히 하나의 행으로 이뤄진 1차원에 정보가 담기게 된다
        • 지역적 정보란 작은 영역 내의 픽셀들이 어떻게 상호작용하는지를 의미
          • 평탄화가 진행되면 이미지 내의 작은 패치에서 색상이나 밝기 변화가 어떤 패턴을 이루고 있는지 알 수 없게된다
    • 또한 추상화과정 없이 바로 연산과정으로 넘어가 버리기 때문에 학습시간과 능률의 효율성 저하
    • 다음과 같은 문제를 해결하기위해 고안한 해결책이 바로 ‘CNN’
    • CNN의 경우 이미지를 날것(raw input) 그대로 받음으로써 공간적/지역적 정보를 유지한 채 특성(feature)들의 계층을 빌드업함
    • CNN의 중요 포인트는 이미지 전체보다는 부분을 보는것과 이미지의 한 픽셀과 주변 픽셀들의 연관성을 살리는 것이다.
  • CNN 사용 이유 예시
    • 어떤 이미지가 랜덤하게 주어졌을 때 새의 이미지인지 아닌지 결정할 수 있는 모델을 개발하기로 가정
    • 이때 새의 이미지 판단 근거로 새의 주요 특징인 ‘새의 부리’가 중요한 근거 포인트가 될 수 있음
    • ‘새의 부리’ 부분은 새의 이미지에서 비교적 작은 부분(지역적인 부분)에 해당함
    • 때문에 모델이 전체 이미지를 보는 것보다 새의 부리 부분 이미지를 잘라 보는게 판단에 더욱 효율적임
    • 이를 위해 사용되는 모델이 바로 CNN이다.
    • 또한 이미지 별로 부리의 위치가 다를 수 있음 따라서 이러한 변수도 고려하여 이미지 부분을 캐치하는 것이 중요함

CNN의 주요 컨셉

  • Convolution의 작동 원리
    • 위와 같은 2차원 이미지는 픽셀 단위로 구성되어 있음
    • 위의 이미지는 MNIST Dataset에서 추출한 샘플중 하나이며 손글씨로 쓰여진 ‘8’의 gray scale 이미지
    • 28x28단위의 픽셀로 구성되어 있으므로 28x28 matrix(행렬)로 표현할 수 있음
    • 즉, 우리는 2차원 이미지를 matrix로 표현할 수 있다 → CNN에 넣어줄 입력값이 바로 이 matrix로 표현된 이미지임
    • 예를 들어 5x5의 input image가 있다고 할때 CNN에는 이미지를 입력 받는 과정에서 활용되는 Filter(Kernel)가 존재함(예시 필터는 3x3크기의 필터)
    • 필터를 사용하여 이미지 입력값을 전체적으로 훑어줌 → 입력값 이미지 모든 영역에 같은 필터를 반복 적용하여 입력값 이미지가 가진 특정 패턴을 찾아 처리하는 것이 목적
    • 그렇다면 필터로 입력값 이미지를 훑어줄 때 어떤 식으로 작용하여 패턴을 찾는 것인가?
    • 내적(Inner product)를 통해 입력값 이미지(matrix)와 필터(matrix)간의 연산처리를 해줌
    • convolution 과정
      • 먼저 입력값 이미지의 빨간 테두리부분과 필터를 내적하여 구한 값을 결과 행렬(result matrix)에 입력한다
      • 다음으로 오른쪽으로 한칸 이동하여 위의 과정을 반복
      • 이런 식으로 입력값 이미지에 필터를 특정 간격으로 이동시켜 가면 내적한 결과값을 결과 행렬(result matrix)에 넣어줌
      • 결과 행렬(result matrix)의 크기는 3x3 → 입력값 이미지(5x5)에 필터(3x3)을 넣고 움직일수 있는 최대 너비와 높이가 3x3이기 때문
  • Zero Padding
    • 위에서 설명한 Convolution 처리를 보면 5x5크기의 이미지가 필터처리를 해주었더니 3x3으로 줄어든 것을 알 수 있음
    • 즉, 손실되는 부분이 발생한다는 뜻 → 이런 손실을 막기위해 ‘Padding’이라는 방법을 사용
    • Padding : 0으로 구성된 픽셀 테두리를 입력값 이미지 가장자리에 감싸주는 방법
      • 픽셀의 값이 0이기 때문에 내적에 영향을 주지 않으면서 필터가 입력값 이미지에서 움직일 수 있는 범위는 늘어나게 하는 목적을 가지고 있음
    • 위의 입력값 이미지에 1만큼 Padding해주면 입력값 이미지의 크기는 7x7로 늘어나게 됨, 이 상태에서 필터를 적용하면 (7-3+1) x (7-3+1) 즉, 5x5 크기의 결과 행렬이 나오게 된다
      • 패딩 계산
        • 새로운 너비 = 원본 너비 + 2 x 패딩 양
        • 새로운 높이 = 원본 높이 + 2 x 패딩 양

Stride

  • Stride는 필터를 얼마만큼 움직여 주는가에 대한 것
  • 위의 예시에서 필터의 Stride값은 1 → 1칸씩 움직였기 때문에
  • Stride-1이 기본값이며 Stride는 1보다 큰 값이 될 수 있음
  • Stride값이 커질 경우 필터가 이미지를 건너뛰는 칸이 커짐을 의미하므로 결과 행렬의 크기는 작아짐

The Order-3 Tensor

  • 입력값이 3차원인 경우 → RGB 컬러이미지는 높이, 너비, 채널(RGB)로 이루어진 3차원 구조를 가짐
  • 이러한 모양은 Order-3 Tensor라고 지칭 (2차원 - 흑백 이미지는 Order-2 Tensor 또는 matrix라고 지칭함)
  • 이러한 3차원 입력값 이미지의 경우 필터 역시 3차원 k_1 x K_2 x K-3 의 크기를 가진 order-3 텐서가 된다.
  • 3차원 입력값에 필터를 적용하여 내적을 구한 결과값의 모양은 matrix가 됨

CNN의 전체적인 네트워크 구조

  • CNN의 구조는 기존의 완전연결계층(Fully-Connected Layer)과는 다르게 구성
  • 완전연결계층(또는 Dense Layer)에서는 이전 계층의 모든 뉴런과 결합되어 있는 Affine계층으로 구현했지만 CNN는 Convolution Layer와 Pooling Layer들을 활성화 함수 앞뒤에 배치하여 만들어 짐

네트워크 구조 파싱

  • 첫 번째 Convolutional Layer
  • 28x28 크기의 입력갑 이미지에 5x5 크기의 필터 10개로 내적하여 convolution 결과값 10개를 생성(필터의 개수가 10개이므로 결과값도 10개)
  • 생성된 결과값에 Activation function(예-ReLU function)을 적용
  • A Convolutional Layer = convolution + activation → 첫 번째 convolution layer 완성
  • 활성 함수(Activation function)란?
    • 참고 자료 : https://medium.com/@snaily16/what-why-and-which-activation-functions-b2bf748c0441
    • 뉴럴 네트워크가 작동하는 기본 원리는 다음과 같다
    • 정보는 입력층을 통해 은닉층을 거쳐 출력된다 이때 입력값에 가중치(weight)를 곱하고 편향(bias)을 추가한 뒤 활성 함수를 거쳐 출력되도록한다.
    • 만약 활성 함수가 없다면 신경망은 단순히 선형회귀모델을 수행하게 된다.
    • 따라서 활성 함수를 추가하여 선형적인 데이터에 비선형성을 더해줘서 더욱 복잡하고 어려운 문제해결이 가능하도록 만들어준다
    • 주요 활성화 함수 (Sigmoid 함수, ReLU (Rectified Linear Unit) 함수, Softmax 함수)
    • 활성화 함수 부분은 간략히 개념만 정리하고 추후 따로 정리할 예정
  • 첫 번째 Pooling Layer
    • Pooling이란?
    • 이전 단계에서 convolution 과정을 통해 많은 수의 결과값(이미지)들을 생성했음
    • 하지만 10개의 필터를 사용했기 때문에 10장의 이미지 결과값이 도출되어 버려 값이 너무 많아지는 문제가 발생
    • 때문에 고안된 방법이 Pooling이라는 과정 → 각 결과값(feature map)의 차원(dimentionality)을 축소해주는 것이 목적 → 즉, correlation이 낮은 부분을 제거하여 결과값의 크기를 축소시키는 과정임
    • 따라서 feature map에 pooling을 적용하니 12x12 크기로 축소되었다

  • 두 번째 Convolution Layer
    • 이번 Convolutional layer에서는 텐서 convolution을 적용 → 첫 번째 convolution layer와 첫 번째 pooling에서 추출된 결과값이 12x12x10(feature map 이미지 장수) 의 order-3 tensor이기 때문
    • 따라서 5x5x10 크기의 텐서 필터 20개를 사용해줌 → 8x8크기를 가진 feature map 20개가 결과값으로 나온다
  • 두 번재 Pooling Layer
    • 위와 같은 방식으로 Pooling 과정을 처리해줌 → 4x4 크기의 feature map 20개를 얻음
  • 평탄화(Flatten → Vectorization)
    • 앞서 얻은 4x4x20의 텐서를 일자 형태의 데이터로 쭉 펼쳐줌
    • 이 과정에서 Flatten 또는 Vectorization이라고 함
    • 1번재 이미지부터 1행부터 마지막 행까지 한 줄로 이어 붙이는 방식으로 진행
    • → 320-dimension을 가진 벡터(Vector) 형태가 된다
  • 그렇다면 왜? 이렇게 1차원 데이터로 변형해도 상관이 없을까? DNN에서는 평탄화하는 과정에서 문제가 발생했는데 왜 CNN은 괜찮은걸까?
    • DNN의 경우 원본 이미지를 그냥 1차원으로 평탄화 하였기 때문에 문제가 발생
    • 그러나 CNN의 경우 1차원으로 평탄화 시키는 것이 원본 이미지가 아닌 convolution과 pooling을 거친 이미지 특성과 관련된 특이점 데이터이기 때문에 문제가 발생하지 않는다
  • Fully-Connnected Layers(Dense Layers)
    • 마지막으로 1차원으로 변환된 입력값을 하나 혹은 하나 이상의 Fully-Connected Layer에 적용시키고 활성 함수(Activation function)을 적용해주면 최종 결과물을 출력하게 된다

매개변수(Parameter)와 하이퍼 매개변수(Hyper-parameter)

  • Parameter : 모델 내부에 있으며 데이터로부터 값이 추정될 수 있는 설정 변수(configuration variable)
  • Hyper-parameter : 모델 외부에 있으며 데이터로부터 값이 추정될 수 없는 설정 변수
  • 딥러닝의 기본은 파라미터들을 최적의 값으로 빠르고 정확하게 수렴하는 것을 목적으로 함
  • 때문에 어떻게 모델의 파라미터들을 최적화 시키냐는 모델 트레이닝의 중요 포인트
  • 또한 모델을 학습시킬 때 좋은 하이퍼 파라미터를 찾는 것 또한 중요 → 하이퍼 파라미터를 튜닝
  • 학습 가능한 매개변수의 수
    • CNN 모델의 학습 가능한 매개변수(trainable parameter)를 위의 예시 사례에서 알아보자
    • 첫 convolution과정에서 5x5크기의 필터 10개를 사용하였으므로 250개의 매개변수가 존재
    • Pooling layer에서는 단순히 크기를 줄이는 개념이므로 매개변수가 존재하지 않음
    • 두번째 convolution 과정에서 5x5x10크기의 텐서 스무개를 사용하였으므로 5,000개의 매개변수가 추가
    • 두개의 Fully-connected layer들에서 각각 매개변수 32,000개, 1000개가 추가되었습니다.
    • 결과적으로 생략한 절편(intercepts)값까지 고려해보면 총 38,250 이상의 학습 가능한 매개변수를 갖게됨
  • 초매개변수(Hyper-parameters)
    • 그렇다면 CNN 모델에서 튜닝 가능한 하이퍼 파라미터는 어떤 것이 있을까?
      • Convolutional layers: 필터의 갯수, 필터의 크기, stride값, zero-padding의 유무
      • Pooling layers: Pooling방식 선택(MaxPool or AvgPool), Pool의 크기, Pool stride 값(overlapping)
      • Fully-connected layers: 넓이(width)
      • 활성함수(Activation function): ReLU(가장 주로 사용되는 함수), SoftMax(multi class classification), Sigmoid(binary classification)
      • Loss function: Cross-entropy for classification, L1 or L2 for regression
      • 최적화(Optimization) 알고리즘과 이것에 대한 hyperparameter(보통 learning rate): SGD(Stochastic gradient descent), SGD with momentum, AdaGrad, RMSprop
      • Random initialization: Gaussian or uniform, Scaling
