Gradient Descent

  • 경사 하강법은 최적화 알고리즘 중 하나로, 손실함수(Loss Function)의 값을 최소화하는 파라미터(가중치-weight, 편향-bias)를 구하기 위해 손실함수에 대한 각 매개변수의 기울기를 이용하는 방법
  • 기울기가 0인 기점인 손실함수의 값이 최저인 곳이므로 그 지점에 도달하기 위해 매개변수는 기울기의 반대방향으로 움직여야 함
  • 따라서 경사하강법에서 미분값의 - 만큼 빼주는 값으로 가중치를 업데이트 하는 것(편향에 대해서도 동일하게 적용됨)

미분(differentiation)

  • 미분은 볌수의 움직임에 따른 함수값의 변화를 측정하기 위한 도구로 최적화에서 많이 사용되는 기법
  • python에서는 sympy.diff를 이용해 미분을 계산할 수 있다.
import sympy as sym
from sympy.abc import x
sym.diff(sym.poly(3*(x**2) + 5*x + 1), x) # 다항함수를 x로 미분하라
# Poly(6𝑥+5,𝑥,𝑑𝑜𝑚𝑎𝑖𝑛=ℤ)

미분의 의미

  • 미분은 함수의 주어진 점 (x, f(x))에서의 접선의 기울기를 구하는 과정
  • 한 점에서의 기울기를 알면 어느 방향으로 움직여야 함수값이 증가 또는 감소 하는지 알 수 있음
  • 2차원에서는 그래프만 봐도 방향을 파악할 수 있지만 고차원으로 갈 수록 그래프를 그리기도 힘들기 때문에 미분값을 구함으로 값의 변경 방향을 파악
  • 미분값에 따른 데이터 변화

미분의 활용

  • 미분값을 더하는 것을 경사상승법(Gradient Ascent)라고 하며 함수의 극대값 위치를 구할 때 사용
  • 미분값을 빼는 것을 경사하강법(Gradient Descent)라고 하며 함수의 극소값 위치를 구할 때 사용
  • → Full Batch Gradient Descent(BGD - 배치 경사 하강법)
  • 경사상승법, 경사하강법은 극값에 도달하면 미분값이 0이므로 더이상 업데이트가 되지 않아 움직임을 멈춘다 즉, 목적함수 최적화가 자동으로 끝났음을 의미
'''
Input: gradient, init, lr, eps
Output: var
'''
# gradient : 미분을 계산하는 함수
# init : 시작점, lr : 학습률, eps : 알고리즘 종료 조건
var = init
grad = gradient(var)
# 컴퓨터로 계산할 때 미분이 정확히 0이되는 것은 불가능하다.
# 따라서, 충분히 작을 때(eps보다 작을때) 종료하는 조건이 필요하다.
while(abs(grad) > eps):
    # lr은 학습률로, 미분을 통해 업데이트하는 '속도'를 조절한다.
    var = var - lr * grad
    # 종료조건이 성립하기 전까지는 미분값을 계속 업데이트한다.
    grad = gradient(var)

Gradient 벡터

  • 입력값이 이차원 공간의 점이 아닌 n차원 공간의 점인 벡터일 경우 그래프를 따라 왼쪽 오른쪽으로만 이동하지 않고 n차원 만큼 굉장히 많은 방향으로 이동할 수 있게 됨
  • 따라서 이 경우 단순한 미분으로는 함수값의 변화를 측정하기 어려움
  • 이처럼 벡터가 입력값인 다변수 함수의 경우 편미분(partial differentiation)을 사용
  • 편미분 역시 sym.diff호 가능함 → x,y 등의 변수를 여러 개 사용한 다변수 함수(sim.poly)를 집어넣으면 된다.
import sympy as sym
from sympy.abc import x,y
sym.diff(sym.poly(3*(x**2) + 5*x*y + 2*y + 1) + sym.cos(2*x + 2*y), x)
# 6𝑥+5𝑦−2sin(2𝑥+2𝑦)
  • 각 변수별로 편미분을 계산한 gradient 벡터를 경사하강법/경사상승법에 이용할 수 있음
  • 위의 식에서 삼각형을 뒤집어놓은 모양의 기호(∇)를 ‘nabla’라고 하는데 f’(x)대신 벡터∇를 이용해서 변수를 동시에 업데이트 할 수 있음
    • 등고선으로 표현
    • gradient 벡터 ∇f(x,y)는 각 점(x,y)에서 가장 빨리 증가하는 방향과 같다
    • gradient 벡터 -∇f는 ∇(-f)와 같고 이는 각 점 (x,y)에서 가장 빨리 감소하는 방향과 같다

변수가 벡터일 때의 경사하강법

  • 변수가 벡터로 바뀌었을 때, 경사하강법의 알고리즘은 종료 조건 하나만 바뀌면 됨
# 경사하강법 with 벡터
'''
Input: gradient, init, lr, eps
Output: var
'''
# gradient : "Gradient 벡터"를 계산하는 함수
# init : 시작점, lr : 학습률, eps : 알고리즘 종료 조건
var = init
grad = gradient(var)
# 절대값(abs) 대신 norm을 계산해서 종료 조건을 설정한다.
while(norm(grad) > eps):
    var = var - lr * grad
    grad = gradient(var)

경사하강법이 수렴하지 않을 때 - 확률적 경사하강법(SGD)

경사하강법 종류(BGD, SGD, MGD)

배치 경사 하강법(Batch Gradient Descent, BGD)

  • 배치 경사 하강법은 경사 하강법의 손실 함수의 기울기 계산에 전체 학습 데이터 셋의 크기와 동일하게 잡는 방법(경사 하강법 대상의 크기 = 배치 크기)
  • 데이터셋 모두를 대상으로 하다보니 파라미터가 한번 이동할 때마다 계산해야할 값이 지나치게 많으므로 계산 시간도 엄청 길어지고 소모되는 메모리도 엄청나다
  • mini batch 안 모든 데이터를 대상으로 경사 하강법을 실시하므로 안정적으로 수렴한다
  • 안정적으로 수렴하므로 수렴까지 발생하는 총 파라미터 업데이트 수는 매우 적다
  • 그러나 지역 최소해(Local Minimum)에 빠지더라도 안정적으로 움직이므로 빠져나오기 힘들다 즉, Lcal Optima 문제가 발생할 가능성이 높다
  • 또한 학습 데이터 셋이 커질수록 시간과 리소스(메모리 등)의 소모가 지나치게 커진다

확률적 경사 하강법(Stochastic Gradient Descent, SGD)

  • 전체 훈련 데이터셋을 대상으로 학습하는 것은 한정된 리소스를 가지고 있는 우리의 분석 환경에서 매우 비효율적이며, 파라미터 업데이트 수가 적다는 것은 랜덤하게 뽑힌 시작 위치의 가중치 수도 적으므로 Local minimum 현상이 발생할 확률도 높다는 것 → 배치 경사 하강법의 문제점
  • 이를 해결하기 위해 학습 데이터셋에서 무작위로 한 개의 샘플 데이터를 추출하고 그 샘플에 대해서만 기울기를 계산한것이 확률적 경사 하강법이다
  • 샘플 데이터셋에 대해서만 경사를 계산하므로 매번 반복에서 다뤄야할 데이터 수가 매우 적어 학습 속도가 매우 빠르다
  • 또한 하나의 샘플만 대상으로 경사를 계산하므로 매모리 소모량이 매우 낮으며 매우 큰 훈련 데이터 셋이라 할지라도 학습이 가능하다
  • 그러나 무작위 추출된 샘플에 대해서 경사를 구하므로 배치 경사 하강법보다 훨씬 불안정하게 움직인다.
  • 손실 함수가 최소값에 다다를 때까지 위아래로 요동치며 움직이다 보니 학습이 진행되다 보면 최적해에 매우 근접하게 움직이긴 하겠으나 최적해(Global minimum)에 정확히 도달하지 못할 가능성이 있다
  • 지역 최소값(Local minimum)에 빠지더라도 쉽게 빠져나올 수 있으며 그로 인해 전역 최소값(Global minimum)을 찾을 가능성이 BGD에 비해 더 높다
  • 속도가 매우 빠르고 메모리를 적게 사용한다는 장점이 있으나 경사를 구할 때 무작위성을 띄므로 지역 최소값의 탈출은 쉽지만 전역 최소값에 다다르기 힘들다는 단점이 있다

미니 배치 경사 하강법(mini-Batch Gradient Descent)

  • 앞서 이야기한 배치 하강법이나 확률적 경사 하강법은 모두 배치 크기가 학습 데이터 셋 크기와 동일하였으나 미니 배치 경사하강법은 배치 크기를 줄이고 확률적 경사 하강법을 사용하는 기법이다.
  • 예시) 학습데이터가 1000개고 batch size를 100으로 잡았다고 할때 총 10개의 mini batch가 나오게 된다 → 이 mini batch 하나당 한번씩 SGD를 진행, 1 에포크(epoch : 전체 데이터 셋이 학습 완료하는 횟수)당 총 10번의 SGD를 진행한다고 할 수 있다.
  • 일반적으로 우리가 부르는 확률적 경사 하강법(SGD)은 실제론 미니 배치 경사 하강법이다
  • 전체 데이터셋을 대상으로 한 SGD보다 파라미터 공간에서 Shooting이 줄어들게 되는데 이는 한 미니 배치의 손실 값 평균에 대해 경사 하강을 진행하기 때문
  • 그로 인해 최적해에 더 가까이 도달할 수 있으나, Local optima 현상이 발생할 수 있음 → SGD의 장점을 이용하여 학습량을 늘리는 방식으로 해결
  • 배치 크기는 총 학습 데이터셋의 크기를 배치 크기로 나눴을 때 딱 떨어지는 크기로 하는 것이 좋다 → 만약 1050개의 데이터를 100개로 배치 크기를 나누면 마지막 50개의 데이터 셋에 과도한 평가가 이뤄질 수 있음 → 따라서 이런경우 50개는 버림(버릴 50개는 완전 무작위로 선정)
